# 家猫カードジェネレーター「またたび会議」Discord連携 セットアップ手順

図鑑・診断結果の「🌿 またたび会議」で押した**投票**と**改善アイデア**を、チーム1のDiscordに**匿名**で流します。
Discord Webhook は中継（GAS）の中に隠すので、公開する index.html に秘密は出ません。

所要時間：約10分。

---

## 全体の流れ

```
ブラウザ（家猫アプリ）
   │ 投票・アイデアをPOST
   ▼
GAS中継（doPost）  ← Discord Webhook をここに隠す
   │ 整形して転送
   ▼
Discord チャンネル（team1）  ← ここでリアクション投票・スレッド議論
   （任意）スプレッドシートにも記録
```

投票の集計と議論は **Discord側のリアクション・スレッド** で行います（静的サイトはDiscordを読めないため）。
採用したいアイデアが決まったら、`家猫_データソース.csv` の該当タイプの「チュール／またたび（改善策）」列を書き換えてcommit → アプリに即反映されます。

---

## 手順①：Discord Webhookを作る

1. 対象チャンネル（例：#team1）の歯車 → **連携サービス** → **ウェブフック** → **新しいウェブフック**
2. 名前を「家猫のまたたび会議」等にして **ウェブフックURLをコピー**

## 手順②：GAS中継を作る

1. https://script.google.com/ → **新しいプロジェクト**
2. 同梱の `家猫診断_Discord中継_Code.gs` の中身を全部コピペ
3. 左の歯車 **プロジェクトの設定 → スクリプト プロパティ** に追加：
   - `DISCORD_WEBHOOK` ＝ 手順①でコピーしたURL
   - （任意）`SHEET_ID` ＝ 記録用スプレッドシートのID（URLの `/d/ここ/edit` の部分）
4. **デプロイ → 新しいデプロイ → 種類「ウェブアプリ」**
   - 実行するユーザー：**自分**
   - アクセスできるユーザー：**全員**
5. 発行された **ウェブアプリURL（…/exec）をコピー**

## 手順③：アプリに中継URLを設定

**A. 自分だけで試す場合**
アプリ下部「🗣 またたび会議（Discord連携）設定」を開き、
- GAS中継URL に②-5のURLを貼る
- （任意）ニックネーム、Discordチャンネルの表示URL（「💬Discordで話す」ボタン用）
- 「保存」

**B. 公開して全員に使ってもらう場合（推奨）**
`index.html` を開き、先頭付近の
```js
const DISCORD_RELAY="";
```
の `""` の中に②-5のGAS中継URLを貼って保存 → GitHubにcommit。
（設定Bなら、訪問者は何も設定せずに投票・投稿できます）

---

## 動作確認

1. アプリの「猫図鑑」で好きな猫をクリック → 詳細下の「🌿 またたび会議」
2. 「👍 効きそう」を押す → Discordチャンネルに投票が届けばOK
3. アイデア欄に一言書いて「🌿 アイデアを送る」→ Discordに届けばOK

## 運用のコツ

- **投票**：Discordの投稿にチーム側で 👍/🤔/🙅 のリアクションを付けて集計代わりに。
- **議論**：気になる投稿は「スレッドを作成」して深掘り。
- **採用**：良いまたたび案が固まったら `家猫_データソース.csv` を更新（`家猫_データ編集.html` で編集可）→ commit。図鑑・診断に反映されます。

## プライバシー

- 送信するのは「タイプNo・タイプ名・現在のチュール/またたび・投票 or アイデア本文・ニックネーム（任意）・時刻」のみ。個人を特定する情報は送りません。
- ニックネーム未入力時は「名無しねこ」で送られます。
