# 家猫 共有API セットアップ手順（全端末で共有 ＋ Discord連携）

これを正しく設定すると：
- 診断で生まれた猫が、**どの端末からでも**みんなの「🗺 家猫マップ（ienekomap.html）」に出る
- 「🌿 またたび会議」の投票・アイデアが**全員で共有**され、集計バーが増える
- （任意）その投票・アイデアが**Discordにも自動投稿**される

かかる時間：約10分。無料。Googleアカウントだけ必要。

---

## いちばん大事な3つ（つまずきポイント）
1. ウェブアプリのアクセス権は必ず **「全員」**（自分だけ、にしない）
2. コードを直したら **必ず再デプロイ**（後述の「新バージョン」）
3. Discordに流すには **`DISCORD_WEBHOOK`** をスクリプトプロパティに入れる

---

## ステップ1：土台を作る
1. ブラウザで Googleスプレッドシートを新規作成（名前は「家猫データ」など何でも可）
2. 上部メニュー **拡張機能 → Apps Script** を開く
3. エディタの中身を全部消して、同梱 **`家猫_共有API_Code.gs`** の中身を丸ごと貼り付け → 💾保存

## ステップ2：（Discordに流すなら）Webhookを登録
1. Discordで対象チャンネルの ⚙ → **連携サービス → ウェブフック → 新しいウェブフック** → **URLをコピー**
2. Apps Scriptの左メニュー **⚙ プロジェクトの設定 → スクリプト プロパティ → プロパティを追加**
   - 名前：`DISCORD_WEBHOOK`
   - 値：さっきコピーしたURL
   - → **保存**
   （Discordに流さないなら、この手順は飛ばしてOK）

## ステップ3：公開（デプロイ）
1. 右上 **デプロイ → 新しいデプロイ**
2. 種類（⚙のマーク）→ **ウェブアプリ**
3. 設定：
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**★ 全員 ★**  ← ここ超重要
4. **デプロイ** → 権限の確認が出たら、自分のアカウントを選び「許可」
5. 出てきた **ウェブアプリ URL（末尾が `/exec`）をコピー**

## ステップ4：アプリに接続（もう貼ってあります）
`index.html` と `ienekomap.html` の先頭付近：
```
const SHARED_API="https://script.google.com/macros/s/.../exec";
```
ここに **ステップ3でコピーしたURL** が入っていればOK。
（今バージョンには先日いただいたURLを既に埋め込み済みです。もしURLが変わった場合だけ差し替えてください。）

---

## ステップ5：ちゃんと繋がったか確認（1分）
ブラウザのアドレスバーに、あなたの exec URL に続けて入力して開いてください。

1. `…/exec?what=cats`
   → `[]`（まだ猫ゼロ）や `[{...}]` のような**JSON**が出れば **読み取りOK**
   → もし「アクセス権が必要」等の画面が出たら、ステップ3の「全員」ができていません
2. `…/exec?action=vote&no=1&name=テスト&vote=%F0%9F%91%8D`
   → `{"ok":true}` が出れば **書き込みOK**（Discordを設定していれば、この瞬間にチャンネルへ投稿されます）
3. アプリを開くと、画面下に **「🟢 共有サーバに接続OK（みんなの猫 N匹）」** と表示されます。
   ienekomap.html の上部にも **🟢/🔴** の状態が出ます。

🔴が出るときは、ほぼ次のどれかです：
- アクセス権が「全員」でない → ステップ3をやり直し
- コードやプロパティを変えたのに再デプロイしていない → 下記
- URL（/exec）が古い／別のデプロイのもの → 最新のexec URLを貼り直す

## コードやWebhookを直したら（再デプロイ）
**デプロイ → デプロイを管理 → ✏（編集）→ バージョン＝「新バージョン」→ デプロイ**
※この方法ならURLは変わりません。新規デプロイを作るとURLが変わるので注意。

---

## データはどこに貯まる？（プライバシー）
- スプレッドシートに自動で2枚のシートができます：
  - `cats`：ts, cid, x, y, quad, typeName, catname, persona, kit, holo
  - `meeting`：ts, cid, kind(vote/idea), no, name, catname, vote, idea, nick
- 氏名などの個人情報は含みません（cid は端末ごとのランダムID）。

## 採用したまたたび案を反映するには
`meeting` シートで良い案を選ぶ → `家猫_データソース.csv` の該当タイプの「チュール／改善策（＝またたび）」を書き換え → GitHubにcommit。図鑑・診断・マップの詳細に反映されます（編集は `家猫_データ編集.html` が便利）。
