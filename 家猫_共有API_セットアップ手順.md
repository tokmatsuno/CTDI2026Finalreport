# 家猫 共有API セットアップ手順（全端末で共有）

これを設定すると、**端末が違っても**：
- 診断で生まれた猫が、全員の「🗺 家猫マップ」に表示される
- 「🌿 またたび会議」の投票・アイデアが、全員で共有・集計される
- （任意）会議の投稿をDiscordにも自動転送できる

バックエンドは Google スプレッドシート＋GAS（無料・アカウント追加不要）。所要 約10分。

---

## 仕組み

```
みんなの端末（index.html / 家猫マップ.html）
   │  POST（猫・投票・アイデアを送る）   GET（全員ぶんを読む）
   ▼
GAS 共有API（1つ）
   ▼
Google スプレッドシート（cats / meeting シートに自動蓄積）
   （任意）→ Discord にも転送
```

---

## 手順

### ① スプレッドシートとGASを用意
1. Google スプレッドシートを新規作成（名前は「家猫データ」など）
2. メニュー **拡張機能 → Apps Script**
3. 同梱の **`家猫_共有API_Code.gs`** の中身を全部コピペして保存
4. （Discordにも流したい場合のみ）左の歯車 **プロジェクトの設定 → スクリプト プロパティ** に
   `DISCORD_WEBHOOK` = DiscordのWebhook URL を追加
   ※Webhookは Discordの対象チャンネル → 連携サービス → ウェブフック で作成

### ② ウェブアプリとして公開
1. 右上 **デプロイ → 新しいデプロイ**
2. 種類：**ウェブアプリ**
3. 実行するユーザー：**自分**／アクセスできるユーザー：**全員**
4. 発行された **ウェブアプリURL（…/exec）をコピー**
5. 初回は権限承認を求められるので許可

### ③ アプリに共有APIを設定（全員に効かせる＝推奨）
公開する2ファイルの先頭付近に、同じURLを貼ってGitHubにcommit：
- `index.html` → `const SHARED_API="";` の `""` の中
- `家猫マップ.html` → `const SHARED_API="";` の `""` の中

> 自分の端末だけで試すなら、アプリ下部「🗣 みんなで共有（共有API）設定」にURLを貼るだけでもOK（ブラウザごとの保存）。

---

## 動作確認
1. 診断で猫を生成 → 「🗺 みんなの家猫マップを見る」→ 点が増えていればOK
2. 別のスマホ/PCからも診断 → 同じマップに反映されればOK
3. 図鑑の猫で「🌿 またたび会議」→ 投票 → 集計バーが増える／別端末でも同じ集計が見える

---

## 出てくるデータ（プライバシー）
- cats シート：ts, cid(端末の匿名ID), x, y, quad, typeName, catname, persona, kit, holo
- meeting シート：ts, cid, kind(vote/idea), no, name, catname, vote, idea, nick
- いずれも**個人を特定する情報は含みません**。cid は端末ごとのランダムID（氏名ではありません）。

## 補足
- 読み取り（GET）は「アクセス：全員」で動きます。書き込み（POST）はブラウザからの匿名送信です。
- 荒らし対策が必要になったら、GAS側で簡単な合言葉チェックを足せます（必要なら対応します）。
- 採用したいまたたび案が決まったら、`家猫_データソース.csv` の該当タイプを更新→commitで図鑑・診断に反映されます。
